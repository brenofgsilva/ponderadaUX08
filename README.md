## Teste de Conteúdo da Interface — Sistema de Ocorrências

## 1. Tela(s) Analisada(s)

As três telas analisadas pertencem a um sistema de gestão de ocorrências de incêndio/emergência em fazendas.

- Tela 1: página inicial com histórico de ocorrências.
  ![Tela 1 - Página inicial com histórico de ocorrências](assets/tela%201.png)

- Tela 2: cadastro de locais (UPs, fazendas e núcleos).
  ![Tela 2 - Cadastro de locais](assets/tela%202.png)

- Tela 3: gerenciamento de recursos individuais (aviões, caminhões, drones etc.).
  ![Tela 3 - Gerenciamento de recursos individuais](assets/tela%203.png)

## 2. Tipo de Teste

**Tipo:** Conteúdo

Será testado se os rótulos, termos técnicos e hierarquia de informações presentes nas telas são compreendidos corretamente pelo usuário-alvo, sem necessidade de explicação prévia.

## 3. Conjunto de Perguntas (Funil)

### Pergunta 1 — Aberta / Exploratória

"Ao olhar para esta tela pela primeira vez, o que você entende que este sistema faz? Descreva com suas próprias palavras."

### Pergunta 2 — Intermediária / Contexto

"Na página inicial, você vê os números '5 Em Andamento' e '4 Ocorrências Concluídas'. O que você entende por cada um desses estados? O que diferencia uma ocorrência 'em andamento' de uma 'concluída'?"

### Pergunta 3 — Intermediária / Terminologia

"Na tela de cadastro de locais, aparecem os termos UP, Fazenda, Área (ha), Núcleo e UF. Sem que ninguém explique, você consegue identificar o que cada campo significa? Algum termo causou dúvida?"

### Pergunta 4 — Específica / Ação

"Na tela de recursos, você vê categorias como 'Bomba Costal', 'Caminhão Pipa' e 'Drone'. Se precisasse adicionar um novo recurso disponível no Núcleo CS2, quais informações você acha que precisaria preencher — e o que cada campo te pede?"

### Pergunta 5 — Fechada / Verificação

"O botão 'Arquivar' na lista de ocorrências — o que você acha que acontece com a ocorrência ao clicar nele? Ela é excluída, pausada ou outra coisa?"

## 4. Objetivo do Teste

Descobrir se os termos técnicos do domínio (UP, núcleo, recursos individuais, arquivar) e a hierarquia de informações das telas são auto-explicativos para usuários com diferentes níveis de familiaridade com o sistema.

## 5. Ação ou Entendimento Esperado

O usuário deve conseguir compreender o propósito de cada tela, identificar o significado dos campos principais e distinguir ações irreversíveis (excluir) de ações reversíveis (arquivar) apenas pela leitura do conteúdo da interface, sem necessidade de treinamento prévio.
